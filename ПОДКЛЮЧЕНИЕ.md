## Шаги для подключения

### 1. Переменные окружения на сервере

На сервере, где собирается и запускается приложение, создайте `.env.local` (или задайте переменные в панели хостинга):

```bash
NEXT_PUBLIC_VK_APP_ID=54511363
NEXT_PUBLIC_APP_URL=https://spindate.lemnity.ru
```

Без `NEXT_PUBLIC_VK_APP_ID` оплаты в VK работать не будут (остальное приложение — будет).

### 2. Сборка и запуск

```bash
npm ci
npm run build
npm start
```

В проде через PM2 порт задаётся в `ecosystem.config.cjs` (по умолчанию **3001**). Nginx должен проксировать на **тот же** порт. Настройте reverse proxy так, чтобы домен **https://spindate.lemnity.ru** вёл на этот порт.

### 3. Настройки приложения ВКонтакте

1. Откройте [Управление приложениями](https://vk.com/apps?act=manage).
2. Выберите приложение с ID **54511363** (или создайте мини-приложение и подставьте его ID в `.env`).
3. В разделе **«Настройки»** → **«Адрес приложения»** укажите:
   ```
   https://spindate.lemnity.ru
   ```
4. Сохраните настройки.
5. При необходимости включите **VK Pay** в настройках приложения (для оплат в игре).

### 4. Проверка

- Откройте приложение по ссылке из ВК (например, из каталога или по прямой ссылке на мини-приложение).
- Проверьте вход через «Войти через VK», кнопки «Добавить друзей» и покупок (если включён VK Pay).

## Если приложение не открывается во ВКонтакте

- Убедитесь, что по адресу **https://spindate.lemnity.ru** отдаётся именно это приложение и сертификат HTTPS валидный.
- Проверьте, что в настройках приложения ВК указан именно этот URL без лишнего слеша в конце.
- Если хостинг добавляет свои заголовки (X-Frame-Options, CSP), разрешите встраивание с `https://vk.com` и `https://vk.ru` (в проекте уже задано в `next.config.mjs` для Next.js).

После выполнения этих шагов приложение готово к использованию и отправке на модерацию ВК.
