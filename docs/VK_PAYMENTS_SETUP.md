# Настройка платежей VK (голоса) для Spindate

По [документации VK](https://dev.vk.com/ru/mini-apps/settings/payments/setting-up): мини-приложения могут продавать виртуальные ценности за голоса ВКонтакте. Чтобы платежи работали, нужно настроить сервер и панель приложения.

## 1. URL для платёжных уведомлений ВКонтакте

В панели управления приложением:

1. Перейдите в **Мои приложения** → выберите мини-приложение Spindate.
2. В меню слева: **Платежи** → **Подключение**.
3. В поле **«URL для платёжных уведомлений ВКонтакте»** укажите:

   **`https://spindate.lemnity.ru/api/vk/payments`**

   (если приложение открывается по другому домену — замените на ваш домен, путь должен быть `/api/vk/payments`.)

На этот URL ВКонтакте отправляет POST (как правило `application/x-www-form-urlencoded`):

- **get_item** — запрос информации о товаре (название, цена). Сервер отвечает JSON с полем **`response`** (см. [обзор уведомлений](https://dev.vk.ru/ru/api/payments/notifications/overview)).
- **order_status_change** — изменение статуса заказа. Успешный ответ содержит **`response`: `{ "order_id": <число> }`**.

Без этого URL платежи за голоса в настройках приложения работать не будут.

## 2. Защищённый ключ и проверка подписи (`sig`)

В настройках приложения указан **защищённый ключ** (secret). Тот же ключ нужно задать на сервере в переменной **`VK_PAYMENTS_SECRET`** (см. `.env.example`).

Алгоритм подписи совпадает с описанием в [документации VK](https://dev.vk.ru/ru/api/payments/notifications/overview) и эталонной реализацией (например, пакет `payments` в [vksdk](https://github.com/SevereCloud/vksdk)): из параметров запроса (без `sig`) собирается строка `ключ=значение` в алфавитном порядке имён параметров, к ней дописывается secret, от результата берётся **MD5 в hex**.

Если `VK_PAYMENTS_SECRET` **задан**, запросы с неверной подписью отклоняются (код ошибки **10**). Если переменная **не задана**, подпись не проверяется (удобно для локальной отладки, **не рекомендуется для продакшена**).

## 3. Товары в приложении (каталог)

Серверный каталог задаётся в `app/api/vk/payments/route.ts` и должен совпадать с тем, что передаётся в **`item`** при вызове `VKWebAppOpenPayForm` (см. `lib/vk-bridge.ts`, `VK_ITEM_IDS`).

Цены в голосах для пакетов сердец считаются через `lib/heart-shop-pricing.ts` (`payVotesForPack`).

| item_id       | Название (пример) | Цена (голоса) |
|---------------|-------------------|---------------|
| `hearts_5`    | 5 сердец          | 1             |
| `hearts_50`   | 50 сердец         | 3             |
| `hearts_150`  | 150 сердец        | 9             |
| `hearts_500`  | 500 сердец        | 30            |
| `hearts_1000` | 1000 сердец       | 50            |
| `hearts_5000` | 5000 сердец       | 270           |
| `vip_7d`      | VIP 7 дней        | 20            |
| `vip_30d`     | VIP 30 дней       | 70            |

Для **get_item** идентификатор товара приходит в параметре **`item`** (как в клиенте).

Подарки в чате, если вызывается оплата **без** `item_id`, могут не проходить через каталог `get_item` — для строгого соответствия сценарию «товар из каталога» передавайте в `showPaymentWall` второй аргумент с тем же `item_id`, что в каталоге (или расширьте каталог отдельными позициями).

## 4. Тестировщики платежей

Пока приложение не опубликовано в каталоге, платить за товары могут только пользователи из списка **«Тестировщики платежей»** в настройках приложения (Платежи → Подключение). Добавьте туда свой аккаунт и тестовые аккаунты.

В тестовом режиме к `notification_type` добавляется суффикс **`_test`** (например `get_item_test`). Текущий endpoint обрабатывает такие уведомления так же, как боевые.

Для тестировщиков голоса при оплате не списываются (тестовый режим).

## 5. Версия API

В разделе **Платежи → Подключение** можно указать версию API платёжных уведомлений. Мини-приложения, зарегистрированные с января 2024 года, используют формат **5.132**. Endpoint поддерживает типы `get_item` и `order_status_change`.

## 6. Регистрация в кабинете выплат

Чтобы получать вознаграждение с продаж, нужно зарегистрироваться в [кабинете выплат](https://dev.vk.com/ru/mini-apps/payments/withdrawal). Подробности — в разделе «Вывод средств» документации VK.

## 7. Проверка

После сохранения URL в настройках:

- Откройте приложение в VK и перейдите в магазин.
- Выберите пакет сердец или VIP — должна открыться форма оплаты VK с ценой и названием из **get_item**.
- На сервере в `.env` должен быть задан **`VK_PAYMENTS_SECRET`**, совпадающий с защищённым ключом приложения.

Логи входящих запросов и ответов можно смотреть в панели VK: **Платежи** → **Логи**.

## 8. Как начисляется товар в текущей реализации

- Начисление выполняется на `order_status_change` в `app/api/vk/payments/route.ts`.
- Для `hearts_*` сервер увеличивает `voice_balance` в БД (`user_game_state` для привязанного VK-аккаунта, иначе `vk_user_game_state`).
- Добавлена идемпотентность заказов через таблицу `vk_payment_orders`: повторные callback-и не дублируют начисление.
- Если callback пришёл с нефинальным статусом, заказ сохраняется, а начисление выполняется только при финальном статусе (`chargeable`/`success`/`paid`/`charged`).
