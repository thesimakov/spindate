# Публикация мини-приложения во ВКонтакте

Инструкция по подключению оплат, приглашению игроков и масштабированию.

> **Быстрый старт:** пошаговое подключение к VK — в корне проекта: [ПОДКЛЮЧЕНИЕ.md](../ПОДКЛЮЧЕНИЕ.md).

---

## 1. Регистрация приложения и настройка

1. **Создание приложения**  
   [vk.com/apps?act=manage](https://vk.com/apps?act=manage) → «Создать приложение» → «Мини-приложение».

2. **Адрес приложения**  
   Укажите URL развёрнутого приложения. Для этого проекта: **`https://spindate.lemnity.ru`**.  
   Для разработки можно использовать ngrok/локальный туннель с HTTPS.

3. **Переменные окружения**  
   Скопируйте `.env.example` в `.env.local` и задайте:
   ```bash
   NEXT_PUBLIC_VK_APP_ID=54511363
   ```
   ID приложения берётся в настройках приложения ВК (раздел «Настройки»). Для этого проекта уже задан в `.env.example`.

## 2. Оплаты (VK Pay)

### Что уже сделано в коде

- В проекте подключён **VK Bridge** (`@vkontakte/vk-bridge`).
- В `lib/vk-bridge.ts` реализованы вызовы:
  - **`showPaymentWall(amount)`** — открытие формы оплаты (покупка голосов/товаров).
  - **`buyVip()`** — покупка VIP (по умолчанию через ту же форму с фиксированной суммой).

При запуске внутри VK (по `vk_user_id` / `vk_app_id` в URL) и при заданном `NEXT_PUBLIC_VK_APP_ID` вызывается метод **VKWebAppOpenPayForm**.

### Что настроить в VK

1. В [управлении приложениями](https://vk.com/apps?act=manage) откройте своё приложение.
2. Перейдите в **Платежи** → **Подключение** ([подробнее](https://dev.vk.com/ru/mini-apps/settings/payments/setting-up)).
3. В поле **«URL для платёжных уведомлений ВКонтакте»** укажите:  
   **`https://spindate.lemnity.ru/api/vk/payments`**  
   (или ваш домен + `/api/vk/payments`). Без этого URL запросы get_item и order_status_change не будут обрабатываться.  
   На сервере задайте **`VK_PAYMENTS_SECRET`** (тот же защищённый ключ, что в настройках приложения ВК), иначе проверка подписи уведомлений не включится — см. [VK_PAYMENTS_SETUP.md](VK_PAYMENTS_SETUP.md).
4. При необходимости добавьте **тестировщиков платежей** (до публикации платить могут только они).
5. Подробная инструкция по платежам: [VK_PAYMENTS_SETUP.md](VK_PAYMENTS_SETUP.md).

## 3. Подключение игроков (приглашения)

### Что уже сделано

- Кнопка «Добавить друзей» в магазине вызывает **VKWebAppShowInviteBox** — стандартный диалог ВК «Пригласить друзей».
- Друзья переходят по ссылке в мини-приложение; авторизация через VK (при входе через «Войти через VK») даёт доступ к данным профиля.

### Дополнительно (по желанию)

- **Друзья в игре**: через [VK API friends.get](https://dev.vk.com/method/friends.get) можно получать список друзей и подставлять их в комнаты (нужен серверный доступ с токеном и правами).
- **Расширенная виральность**: короткая ссылка вида `https://vk.com/app12345678` — уже есть у приложения в настройках; её можно использовать в постах и рассылках.

## 4. Масштабирование

### Широкоформатный режим и размер iframe (десктоп)

В десктопной версии сайта VK игра открывается во **iframe**. Ширина колонки с приложением задаётся **в панели приложения**, а не в коде репозитория.

По [официальной документации «Отображение»](https://dev.vk.com/ru/games/settings/general/display#Размер%20iframe):

1. Откройте [управление приложениями](https://vk.com/apps?act=manage) → ваше мини-приложение.
2. В меню слева: **Настройки** → **Отображение** (если пункт называется иначе — раздел про отображение игры / iframe на десктопе).
3. В блоке **«Десктопная версия сайта»** включите **«Широкоформатный режим»**.  
   Тогда ширина iframe **подстраивается под ширину вкладки браузера**. Если режим **выключен**, используется фиксированный **«Размер iframe»** (ширина до **1000 px** — визуально похоже на «планшетную» колонку).
4. Параметр **«Размер iframe»** (`ширина × высота`, минимум 630×600, максимум **1000×4500** px): **высота** задаёт высоту iframe; при включённом широкоформатном режиме **заданная ширина не используется** — важна высота.
5. Полноценный широкоформатный режим для обычных пользователей доступен **после публикации в каталоге**; до этого его видят создатель и администраторы с правом тестирования.

В коде приложения при загрузке вызываются **`VKWebAppInit`**, **`VKWebAppExpand`** и **`VKWebAppSetViewSettings`** (`lib/vk-bridge.ts`, `components/game-app.tsx`) — они помогают развернуть контейнер и применить настройки в доступных клиентах, но **не заменяют** включение широкоформатного режима в настройках VK.

### Текущая архитектура

- **Клиент**: Next.js, состояние игры в React Context (игроки, стол, инвентарь и т.д.).
- **Сохранение**: локально в браузере (при перезагрузке данные теряются).
- **Игроки за столом**: сейчас — пользователь + боты; реальные друзья требуют бэкенда.

### Рекомендации при росте нагрузки

| Задача | Решение |
|--------|--------|
| **Сохранение прогресса (сердечки, розы, инвентарь)** | Бэкенд (Node/Next API Routes или отдельный сервис) + БД (PostgreSQL, SQLite и т.д.). При входе через VK передавать `vk_user_id` и при первом заходе создавать/обновлять запись. |
| **Мультиплеер (реальные игроки за одним столом)** | WebSocket-сервер (Socket.io, uWebSockets) или облачные комнаты (например, на базе VK или сторонних сервисов). Идентификация по `vk_user_id`. |
| **Вебхуки оплат VK** | Отдельный endpoint (например, `POST /api/vk/payment-callback`), проверка подписи, начисление товара в БД и ответ в формате, который ожидает VK. |
| **Статика и CDN** | Деплой на Vercel/Netlify даёт CDN и масштабирование «из коробки». Для тяжёлых медиа — отдельное хранилище (S3, VK Cloud и т.д.). |

### Переменные окружения при масштабировании

```bash
# Обязательно для оплат в VK
NEXT_PUBLIC_VK_APP_ID=54511363

# При появлении бэкенда
DATABASE_URL=postgresql://...
NEXT_PUBLIC_API_URL=https://your-api.example.com

# Секрет для вебхуков VK (из настроек приложения)
VK_PAYMENT_SECRET=...
```

## 5. Соответствие правилам VK Mini Apps (п. 2.3.8)

По [правилам размещения](https://dev.vk.com/ru/mini-apps-rules) в мини-приложениях запрещены азартные игры на реальные деньги. В Spindate:

- **Виртуальная валюта:** в игре используются только «сердечки» (игровая валюта). Они не обмениваются на реальные деньги и не выводятся.
- **Ставки в игре:** блок «Ставка» в игровой комнате — это ставка сердечками на угадывание пары; выигрыш выплачивается только в сердечках. Явно указано в интерфейсе: «Не является азартной игрой на деньги (п. 2.3.8 правил VK Mini Apps)».
- **Магазин:** пополнение сердечков за голоса VK (или тестовые платежи) оформлено как покупка виртуальной валюты для использования внутри приложения; в магазине есть текст о соответствии п. 2.3.8.
- **Регистрация:** при входе пользователь соглашается с правилами размещения VK и видит, что валюта виртуальная и не является азартной игрой на деньги.

Перед отправкой на модерацию убедитесь, что ссылка на правила актуальна: [dev.vk.com/ru/mini-apps-rules](https://dev.vk.com/ru/mini-apps-rules).

## 6. Чек-лист перед публикацией

- [ ] Приложение создано в [vk.com/apps?act=manage](https://vk.com/apps?act=manage), в настройках указан URL: **https://spindate.lemnity.ru**
- [ ] В `.env.local` задан `NEXT_PUBLIC_VK_APP_ID`.
- [ ] Проект собран: `npm run build` и развёрнут по HTTPS.
- [ ] В настройках приложения ВК включены права доступа, необходимые для VK Pay и приглашений (если требуются отдельные права).
- [ ] Проверен вход через «Войти через VK» и кнопки «Добавить друзей» и покупок в интерфейсе приложения внутри VK (в мобильном и десктопном клиенте).
- [ ] В настройках VK (**Настройки → Отображение**) для десктопа включён **широкоформатный режим**, если нужна ширина под вкладку браузера (см. подраздел «Широкоформатный режим и размер iframe» в разделе 4 этого файла).
- [ ] На экране входа отображается ссылка на правила VK и текст о виртуальной валюте; в блоке ставок и в магазине есть пометка о соответствии п. 2.3.8.

После этого приложение можно отправлять на модерацию и публиковать в каталоге мини-приложений ВКонтакте.
